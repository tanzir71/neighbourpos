Local-first POS + CRM for neighborhood stores.

Capture orders, track customers by phone, build simple segments, and export or queue campaigns — offline after setup.

Local-first Offline SQLite PDO Segmentation Export

Workflow

Ingest → sell → learn → target. All local by default.

POS

Search inventory, build carts, attach a phone (or walk-in), and update statuses fast on mobile.

CRM targeting

Customers are keyed by phone. Orders feed recency/frequency/spend so segments stay actionable.

Campaigns

Build segments, preview recipients, simulate impact, then export/queue messages with opt-in-by-default safeguards.

Customer portal

Customers can look up order status via phone (and you can optionally add SMS links later).

Stack

Local-first architecture built for shared hosting.

App
Single-file PHP + SQLite + embedded mobile UI
Database
SQLite (WAL) via PDO
Security
CSRF + CSP + session timeout + rate limiting
Integrations
SMS/Email placeholders (Twilio/SendGrid/etc.)
